# Implementation Documentation

This directory contains phase-specific implementation documentation and testing guides.

## Organization

- **`phase1/`** - Phase 1: Core Infrastructure & Docker Setup
  - Implementation plans, completion summaries, checklists
- **`phase2/`** - Phase 2: TradingNode Integration (to be created)
- **`phase3/`** - Phase 3: External Adapter Framework (to be created)
- **`testing/`** - Testing documentation (shared across phases)
  - Manual testing guides, test plans, test results

## Important Notes

- **SSOT Documents**: Only `../ARCHITECTURE.md`, `../ROADMAP.md`, `../FILE_ORGANIZATION.md`, and `../README.md` are the single source of truth
- **Implementation Files**: These files document the implementation process but are not SSOT
- **Organization**: Keep files organized by phase and purpose
- **Context7**: Always use Context7 for external library documentation (NautilusTrader, etc.)

## File Naming Convention

- Phase-specific files: `PHASE{N}_{PURPOSE}.md` (e.g., `PHASE1_COMPLETION_SUMMARY.md`)
- Testing files: `{PURPOSE}_TEST_{TYPE}.md` (e.g., `MANUAL_TESTING_GUIDE.md`)
- Checklists: `{PURPOSE}_CHECKLIST.md` (e.g., `PRE_PUSH_CHECKLIST.md`)
